# step1:设计默认首页
## 将界面主要划分为三块：1:header 2:main  3:footer
## 将每个划分内的内容进行填充，并在style标签内设计样式，使用flex来进行布局。
### 1 header：包括主标题和副标题
### 2 main：包含四张快捷建议卡片
### 3 footer:主要是提问栏和两个功能按钮。

### 遇到的一些问题：
 1：在点击模式切换时只是会闪一下，点击后还是暗夜模式。原因：button默认是submit，在点击时会触发表单提交，会应该调整为普通button
2：切换模式后，卡片和按钮的背景颜色并没有变化。 因为在style中对于CSS选择器的书写有要求，需要在body的class名与后面的容器名之间留一个空格，这样才是后代选择器，否则意义就变为了在一个标签中同时拥有这两个类才会执行。

# step2:按键功能实现效果

## 1：主题切换：初始为深色模式，点击月亮按钮后会转换为浅色模式，图标变为太阳。
### 过程：在CSS中，对于与颜色有关的项，都会定义两套样式，一个是默认，另一个是在body下新增的一个light类，当body有这个类时，就会执行，把前面的样式覆盖掉，否则不会执行，保留默认样式。在javaScript中，通过按钮触发动作来决定light类的增删，实现对于颜色深浅的控制。

###  问题：当点击刷新按钮时并不会保留之前的模式，都会重置为默认深色模式。
    需要在js中使用localStorage实现数据的存储和读取，当每一次设置好模式后，都会将当前的选择存储到浏览器本地文件，当刷新页面时，首先读取上一次的数据，对界面进行初始化，就实现了刷新后仍保持原模式。

## 2:对话框自适应高度： 希望实现对话框的高度会随着其中内容的变化进行调整，超出最大范围时会固定。
### 过程：首先将对话框固定在页面最下方，设置最小高度和最大高度，以及行高。
### 在javaScript中监听输入事件，计算内容真实的高度，并设置对话框的高度。
### 当输入的内容超出默认显示高度时，输入框会向上扩张。

## 3:有内容时显示发送按钮：当对话框空白时，只有上传图片一个按钮，当有内容时，对话框最右侧会显示发送按钮，同时上传按钮会移动到左边。
### 过程： 首先将提交按钮设置为默认不可见。在css文件中对两个按钮的样式进行重写，重新调整他们的位置和是否可视，还是要用到在js中监听输入事件，当有输入时，给对话框的容器加一个类，当存在这个类时，会执行css中的两个按钮的新样式。

## 4：支持图片上传功能，且图片可预览
### 过程：在html的上传按钮后设置一个默认不可见的上传文件的<input>类型，以及用于存放预览图片的容器，并将上传按钮的点击动作与其绑定，在js中完成对上传图片的读取和预览。
